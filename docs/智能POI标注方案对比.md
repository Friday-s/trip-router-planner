# 智能POI数据标注方案对比

> 目标：找到不需要人工标注的自动化方案
> 状态：方案调研中

---

## 一、可选API方案对比

### 方案A：携程开放平台 API

| 项目 | 说明 |
|-----|------|
| 官网 | https://open.ctrip.com |
| 能获取的数据 | 景点名称、简介、开放时间、门票价格、用户评分、游玩时长、图片 |
| 优点 | 数据最全，游玩时长直接有 |
| 缺点 | 需要企业资质申请，审核周期长，可能有调用限制 |
| 是否有标签 | 有携程自己的分类标签 |
| 经纬度 | 可能需要配合高德转换 |

**携程API能返回的字段**（根据网上资料）：
- 景点名称、地址
- 景点简介（详细描述）
- 开放时间
- 门票价格
- 用户评分
- **推荐游玩时长** ✓
- 用户评论
- 图片

---

### 方案B：百度地图 Place API

| 项目 | 说明 |
|-----|------|
| 官网 | https://lbsyun.baidu.com |
| 能获取的数据 | 名称、地址、经纬度、评分、营业时间、电话、分类 |
| 优点 | 官方API，稳定，有详情接口 |
| 缺点 | 与高德类似，业务标签仍需补充 |
| 是否有标签 | 有百度分类（与高德类似） |
| 经纬度 | 百度坐标系，需转换 |

**百度比高德多的字段**：
- 营业时间（部分景点有）
- 人均消费（餐饮类）
- 详细描述（scope=2时）

---

### 方案C：AI大模型智能标注（推荐）

| 项目 | 说明 |
|-----|------|
| 可选模型 | Claude API / ChatGPT API / 通义千问 / 文心一言 |
| 工作方式 | 把高德获取的基础数据喂给AI，让AI补充业务字段 |
| 优点 | **智能理解**，不是简单标签，可以生成丰富的描述 |
| 缺点 | 有API调用成本，需要校验准确性 |
| 是否需要人工 | 大幅减少，只需抽查校验 |

**AI可以智能生成的内容**：

### 方案D：高德 + 携程 + AI 组合（最佳方案）

```
┌─────────────────────────────────────────────────────────────────┐
│                    三层数据融合方案                               │
└─────────────────────────────────────────────────────────────────┘

Layer 1: 高德地图API
         ↓
         获取：名称、经纬度、地址、高德分类
         作用：提供准确的地理位置数据

Layer 2: 携程/百度API（可选）
         ↓
         获取：评分、开放时间、门票价格、游玩时长
         作用：补充基础运营数据

Layer 3: AI大模型
         ↓
         生成：智能标签、适合人群、游玩建议、亮点介绍
         作用：生成智能化、个性化的描述

最终输出：完整的POI数据
```

## 二、AI智能标注详细方案

### 2.1 工作流程

```
Step 1: 高德API获取基础数据（50个景点）
        ↓
        得到：name, location, address, type, rating

Step 2: 构造Prompt，调用AI API
        ↓
        输入：景点名称 + 地址 + 高德分类
        输出：结构化JSON（标签、时长、亮点、建议等）

Step 3: 合并数据
        ↓
        高德数据 + AI生成数据 = 完整POI数据

Step 4: 抽查校验（可选）
        ↓
        人工检查几个样本，确保AI生成质量
```

### 2.2 AI Prompt 设计

```
你是一个大理旅游专家。我会给你一个景点的基本信息，请你返回该景点的详细标注数据。

输入景点信息：
- 名称：{name}
- 地址：{address}
- 高德分类：{type}

请返回JSON格式，包含以下字段：

{
  "area": "所属区域（古城区/洱海东岸/洱海西岸/喜洲/苍山/双廊/剑川 之一）",
  "primary_tag": "主标签（古镇/自然风光/历史遗迹/民族文化/宗教场所/休闲娱乐 之一）",
  "tags": ["标签数组，2-4个，如：摄影、徒步、亲子、小众、网红打卡"],
  "visit_duration": 游玩时长（分钟，整数）,
  "best_time": "最佳游玩时间",
  "highlights": ["3个主要亮点"],
  "brief": "一句话介绍（30字以内）",
  "tips": "游玩小贴士",
  "suitable_for": ["适合人群，如：情侣、家庭、摄影爱好者"],
  "difficulty": "游玩难度（轻松/适中/有挑战）"
}

只返回JSON，不要其他内容。
```

### 2.4 API成本估算

| 模型 | 价格 | 50个景点成本 |
|-----|------|-------------|
| Claude 3.5 Sonnet | $3/百万token | 约 ¥1-2 |
| GPT-4o | $5/百万token | 约 ¥2-3 |
| 通义千问 | ¥0.008/千token | 约 ¥0.5 |
| 文心一言 | ¥0.012/千token | 约 ¥0.8 |

**结论：成本极低，50个景点约几块钱**

## 三、方案对比总结

| 方案 | 自动化程度 | 数据丰富度 | 实施难度 | 成本 |
|-----|-----------|-----------|---------|------|
| 高德 + 人工 | 低 | 中 | 低 | 人力成本 |

| 高德 + 百度 | 高 | 中 | 低 | 免费 |
| **高德 + AI** | **高** | **很高** | **低** | **几块钱** |

## 四、推荐方案

### 首选：高德API + AI大模型

**理由**：
1. **完全自动化**：无需人工标注
2. **数据智能**：AI可以生成比简单标签更丰富的信息
3. **成本极低**：50个景点只需几块钱
4. **灵活可控**：可以随时调整Prompt优化输出

**实施步骤**：
1. 用高德API获取50个核心景点的基础数据
2. 设计好Prompt模板
3. 批量调用AI API生成智能标注
4. 合并数据，导入数据库
5. 抽查几个样本确认质量

## 五、需要确认的问题

1. **你有AI API的Key吗？**（Claude/ChatGPT/通义千问等）

3. **是否需要我先用AI生成几个样本看效果？**
