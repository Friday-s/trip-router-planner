# 旅游线路规划平台设计与实现

> 基于AI智能推荐的旅游线路规划平台，以云南大理为示范城市，提供智能行程生成、地图可视化、社区分享等功能。

## 项目概述

### 研究背景

随着自由行旅游的普及，游客在规划行程时面临以下痛点：
- 景点信息分散，需要在多个平台间切换查找
- 行程安排缺乏合理性，容易出现路线绕远、时间冲突
- 难以获取真实的本地游玩经验和推荐

### 研究目标

本项目旨在设计并实现一个智能化的旅游线路规划平台，通过集成AI大模型技术，为用户提供一站式的行程规划服务。系统采用可插拔的推荐引擎架构，便于后续扩展本地算法。

### 主要工作

1. **需求分析与系统设计**：分析用户需求，设计系统架构和数据库结构
2. **POI数据采集与标注**：利用高德地图API采集景点数据，通过AI进行智能标注
3. **智能推荐引擎实现**：基于DeepSeek大模型实现智能行程推荐
4. **前后端系统开发**：采用Vue 3 + Spring Boot技术栈完成全栈开发
5. **地图可视化**：集成高德地图JS API实现路线可视化展示

## 核心功能

| 功能模块 | 描述 |
|---------|------|
| AI智能行程生成 | 根据天数、主题偏好、出行节奏自动生成合理的多日游行程 |
| 地图可视化 | 在地图上展示景点和路线，包含步行/驾车/公交耗时估算 |
| 景点浏览 | 按区域、标签筛选景点，查看详细信息 |
| 行程管理 | 保存、查看、删除个人行程 |
| 社区分享 | 用户可发布游记帖子，形成UGC内容生态 |
| 后台管理 | 管理员维护POI数据和社区内容 |

## 快速开始

### 环境要求

- JDK 17+
- Node.js 18+
- MySQL 8.0+
- Redis 7.0+
- Maven 3.8+

### 数据库初始化

```bash
# 创建数据库
mysql -u root -e "CREATE DATABASE trip_planner CHARACTER SET utf8mb4;"

# 导入表结构和初始数据
mysql -u root trip_planner < database/schema.sql
mysql -u root trip_planner < database/init-data.sql
```

### 后端启动

```bash
cd backend

# 配置环境变量（可选，有默认值）
export DEEPSEEK_API_KEY=your-api-key

# 启动应用
mvn spring-boot:run
```

后端服务地址：http://localhost:8080/api

### 前端启动

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端访问地址：http://localhost:3000

## 技术特点

1. **可插拔推荐引擎**：采用策略模式设计RecommendationEngine接口，支持AI引擎和规则引擎切换
2. **AI智能标注**：利用大模型对POI数据进行智能标注，生成游玩建议和标签
3. **前后端分离**：Vue 3 + Spring Boot现代化技术栈
4. **响应式设计**：适配桌面端和移动端浏览器
